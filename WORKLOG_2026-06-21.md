# AlbumPlace 작업 정리

- 작성일: 2026-06-21
- V1 최종 버전: `1.1.6`
- V1 최종 versionCode: `15`
- 상태: V1 개발 종료, Closed Testing/Play Store 배포 준비

## 1. V1 제품 방향

AlbumPlace의 역할을 단순한 파일 정리 도구에서 아래 경험으로 정리했다.

- 위치 기반 사진/동영상 앨범 정리
- 정리 결과에서 과거 장소 발견
- 장소와 날짜를 통한 기억 회상
- 실제 사진 탐색은 Samsung Gallery에 위임

핵심 원칙:

- AlbumPlace는 갤러리 앱을 복제하지 않는다.
- 장소명은 앱이 발견한 실제 앨범명으로 유지한다.
- 사용자가 입력한 기억은 장소명과 별도 정보 계층으로 표시한다.
- 원본 사진은 복사하고, 동영상은 사용자 옵션에 따라 이동한다.
- 원본 사진 삭제는 Android 휴지통 승인 흐름만 사용한다.

## 2. V1 최종 화면 구조

### 하단 탭

1. 홈
2. 정리 기록
3. 설정

### 홈

- 새 장소 / 위치 없음 / 정리 완료 요약
- 앨범 정리 시작 CTA
- 최근 발견한 장소 4개
- 장소 카드에는 장소명, 사진 수, 최근 촬영일 표시
- 정리 대상 폴더 선택은 설정으로 이동

### 정리 기록

- 장소별 2열 카드 목록
- 표시 순서:
  1. 실제 장소명
  2. 기억 한 줄(있는 경우만)
  3. 사진 수
  4. 시작 월~최근 월
- 최근 기록 기준 정렬

### 장소 상세

- 실제 장소명을 상단 제목으로 표시
- 대표 사진
- `장소명의 기록`
- 기억 한 줄(있는 경우만)
- 사진 수와 기간
- 첫 방문 / 최근 방문
- 기억 편집용 연필 아이콘
- 최근 사진 미리보기
- Samsung Gallery 해당 앨범으로 이동

## 3. 오늘 완료한 기능

### 정리 기록 내비게이션

- 정리 기록 목록에서 상세 화면 진입 후 시스템 Back/상단 Back 시 홈이 아닌 목록으로 복귀한다.
- 상세 진입 전 목록의 `ScrollView` 위치를 저장하고 복귀 시 복원한다.
- 홈 이동은 하단 홈 탭을 눌렀을 때만 수행한다.

### Samsung Gallery 앨범 직접 이동

- 정리 기록 상세의 `갤러리에서 보기`가 일반 갤러리 홈이 아닌 해당 앨범을 우선 실행한다.
- 사용한 Samsung Gallery 비공개 액션:
  - action: `com.android.gallery.action.SHORTCUT_ALBUM_VIEW`
  - component: `com.sec.android.gallery3d/com.samsung.android.gallery.app.activity.external.GalleryExternalActivity`
  - `ALBUM_ID`: MediaStore `BUCKET_ID` 정수값
  - `IS_VIRTUAL_ALBUM=false`
  - `key-album-type=0`
- 이미지와 동영상 양쪽에서 `BUCKET_ID`를 조회한다.
- Samsung Gallery가 없거나 직접 실행이 실패하면 안내 후 일반 갤러리를 연다.
- 연결된 Galaxy에서 `송파구에서` 앨범(사진 43개, 동영상 1개) 직접 이동을 확인했다.

### 기억 한 줄

- 정리 기록 하나당 로컬 기억 한 줄을 추가·수정·삭제할 수 있다.
- 연필 아이콘은 상세 화면의 첫 방문 정보 옆에 배치했다.
- 기억은 `SharedPreferences`에 앨범 `relativePath` 기준으로 저장한다.
- 장소명은 변경하지 않는다.
- 기억이 없으면 기억 줄을 숨긴다.
- 중간 구현에서 저장된 표시 이름 값은 기억 값이 비어 있을 때 기억으로 이어받는다.

### 장소/기억 표시 계층

최종 표시 예시:

```text
송파구에서
발레학원
44개 사진
2023.5~2026.6
```

- `(기록)`, `(메모)`, `장소명 - 메모` 형식은 사용하지 않는다.
- 상세 본문은 `송파구에서의 기록` 아래에 `발레학원`을 별도 줄로 표시한다.

### 휴지통 완료 후 결과 정리

- 사용자가 사진 원본의 휴지통 이동을 승인하고 작업이 끝나면 일회성 정리 결과를 닫는다.
- 홈으로 복귀하고 임시 분석/정리 상태를 비운다.
- JSON 기반 정리 기록과 장소 목록은 유지한다.
- 원본 휴지통 처리는 `MediaStore.createTrashRequest()`만 사용한다.

## 4. 날짜 범위 버그 조사와 수정

### 증상

- `Grantville에서`의 실제 사진은 2019년 12월인데 정리 기록이 `2019.12~2020.1`로 표시됐다.
- 일부 다른 장소도 폴더 날짜 범위가 실제 촬영 시점과 달랐다.

### 확인 결과

`Grantville에서`에는 사진 6장이 있었다.

- 4장: 2019-12-28
- 문제의 2장:
  - 파일명/MediaStore 수정일: 2020-01-02
  - MediaStore `DATE_TAKEN`: 없음
  - EXIF `DateTimeOriginal`: 2019-12-28 16:27:54

기존 장소 요약은 `DATE_TAKEN`이 없을 때 수정일을 사용했기 때문에 2020년 1월까지로 계산했다.

### 수정 내용

- 기존 위치 앨범을 요약할 때 `DATE_TAKEN`이 없는 항목만 추가 메타데이터를 읽는다.
- 날짜 보완 우선순위:
  1. EXIF 촬영일 또는 동영상 메타데이터
  2. 파일명에서 파싱한 날짜
  3. MediaStore 수정일
  4. MediaStore 추가일
- 전체 5,176장 중 `DATE_TAKEN`이 없는 49장에만 메타데이터 보완을 수행해 성능 영향을 제한했다.
- 신규 원본 분석에서도 파일명보다 EXIF/동영상 촬영 메타데이터를 우선한다.
- 원본 파일과 MediaStore 값은 수정하지 않고 앱의 표시 계산만 바로잡았다.

### 실기 검증

code15 APK를 연결된 Galaxy에 설치해 확인했다.

- `Grantville에서`: `2019.12~2020.1` → `2019.12`
- `Melbourne에서`: 동일한 보완 경로로 `2019.12` 표시 확인
- 원본 파일 변경 없음

- 결과 화면에서 중복 휴지통 버튼 제거, 앨범 행의 버튼만 유지
- 홈/목록/상세 카드, 아이콘, 여백, 컬러 체계 정리

## 8. 남은 위험과 운영 원칙

- Samsung Gallery 직접 이동은 Samsung 비공개 Intent에 의존하므로 One UI 업데이트에서 동작이 바뀔 수 있다.
- Galaxy 외 기기에서는 일반 갤러리 fallback만 보장한다.
- 날짜 보완은 `DATE_TAKEN`이 없는 항목을 대상으로 한다. 값이 존재하지만 잘못된 외부 파일은 별도 사례 확인이 필요하다.
- 휴지통 완료 후 홈 복귀는 코드/빌드 검증을 완료했지만 추가 원본을 사용한 파괴적 회귀 테스트는 생략했다.
- 자동화된 단위 테스트는 현재 없으며 주요 흐름은 실기 테스트 중심이다.
- V1 출시 후 발견되는 경미한 UI 문제는 기록만 남기고 V2에서 처리한다.
- 원본 파일 날짜, EXIF, MediaStore row를 임의로 수정하는 기능은 추가하지 않는다.

## 9. V2 작업 목록

V2 목표: **정리하면 기억이 보인다(V1) → 기억을 내 것으로 만든다(V2)**

### P0. 기억 기능 확장

V1의 기억 한 줄을 기반으로 다음을 검토한다.

- 입력 UX와 문구 다듬기
- 장소 상세에서 기억의 시각적 우선순위 개선
- 기억이 있는 장소의 재방문 경험
- 필요 시 수정 이력 또는 더 긴 메모 지원 여부 검토
- 검색·동기화·AI 작성은 바로 추가하지 않는다.

### P1. 날짜별 자동 세분화와 추천

- 장소별 사진 분포를 분석해 적절한 단위를 추천한다.
- 추천 기준:
  - 며칠에 집중된 여행: 날짜별
  - 여러 달에 분산: 월별
  - 1년 이상 분산: 연도별
  - 사진 5장 이하: 세분화하지 않음
- 여행 감지 초기 기준:
  - 3~10일 사이에 사진 집중
  - 해당 기간의 하루 평균 사진 수가 평소의 3배 이상
- 상세 화면 예시:

```text
연도별로 나눠봤어요 · 변경
2023년 12개
2024년 18개
2025년 9개
2026년 5개
```

- 사용자가 날짜/월/연도 단위를 장소별로 변경하고 선택값을 로컬 저장한다.

### P1. 상세 화면에서 실제 하위 폴더 정리

- 날짜별·월별·연도별 하위 폴더 생성 기능을 설계한다.
- 예시:

```text
Yufu에서/
  2025-03/
  2025-04/
```

- 추천 화면과 실제 파일 이동은 분리한다.
- 실행 전 변경 내용 미리보기와 명시적 사용자 승인이 필요하다.
- 날짜 보존과 Samsung Gallery 정렬 회귀 테스트를 필수로 한다.

### P2. POI 기반 자동 세분화

- Google Places API 또는 Kakao 로컬 API 후보 비교
- GPS 좌표를 의미 있는 POI로 매핑
- 초기 분리 임계값은 POI당 사진 5장 이상을 검토
- 편의점 지점처럼 과도하게 세분화되는 장소는 제외
- 랜드마크와 반복 방문 장소를 우선

예시:

```text
용인에서 38개
  에버랜드 22개
  기타 용인 16개
```

### P2. 장소 통계와 타임라인

- 연도별 사진 수
- 실제 방문일 수
- 첫 방문 / 최근 방문
- 가장 많이 기록된 장소 순위
- 통계 앱처럼 보이지 않도록 기억 탐색을 보조하는 정보만 노출

### V2에서 하지 않을 것

- 갤러리 전체 사진 브라우저
- 전체 썸네일 그리드 확장
- 앱 내부 사진 슬라이드 뷰어
- 중복 파일 제거
- 위치 없는 사진에 위치 직접 입력
- 자동 하이라이트/스토리 생성
- AI 메모 자동 작성
- 검색·태그·클라우드 백업·동기화
- 실제 Samsung Gallery 앨범명 변경
