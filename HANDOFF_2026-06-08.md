## 오늘 반영한 핵심 UX

### 메인 화면
- 메인 4개 기능 버튼 구조를 제거했다.
- 메인 Primary CTA는 `앨범 정리 시작`.
- 보조문구는 `정리할 사진/동영상을 먼저 확인해요`.
- `바로 정리하기` 버튼은 미리보기 완료 후에만 메인에 표시된다.
- 메인 상단에 `갤러리` 버튼을 추가했다.
- `정리 미리보기` 카드 문구는 개발자스럽지 않게 조정 중.

### 미리보기 팝업
- 팝업 제목: `정리할 항목 확인`.
- `바로 정리하기`를 누르면 실제 정리 실행으로 연결되도록 수정했다.
- `결과 보기`, `갤러리 열기` 버튼은 높이를 줄였다.
- 팝업에서 결과 화면으로 이동한 뒤에도 결과 화면의 `바로 정리하기`로 정리가 가능해야 한다.

### 결과 화면
- 결과 화면의 `바로 정리하기` 버튼은 `정리될 앨범` 섹션 위에 둔다.
- 결과 화면에서 `바로 정리하기`를 누르면 메인 UI를 복원한 뒤 `runCopy()`를 `post`로 실행하도록 수정했다.
- 정리 완료 후 자동으로 결과 화면으로 이동한다.
- 정리 완료 후에는 `바로 정리하기`를 숨기고, `사진 원본 삭제`와 `갤러리 열기` 흐름으로 연결한다.
- `사진 원본 삭제`는 복사된 사진 원본이 있을 때만 결과 화면에 표시한다.
- `갤러리 열기`는 결과 화면에도 유지한다.

## 기능 상태

### 사진
- 위치 기반 앨범으로 복사한다.
- 복사 성공한 원본만 `사진 원본 삭제` 후보가 된다.
- 원본 삭제는 Android `MediaStore.createDeleteRequest`를 사용한다.

### 동영상
- 미리보기 대상에 포함된다.
- 삼성 갤러리 동영상 GPS는 `MediaMetadataRetriever.METADATA_KEY_LOCATION`으로 읽는다.
- 동영상은 복사가 아니라 이동한다.
- 동영상은 원본 삭제 후보에 넣지 않는다.

### 위치명 표시
- 실제 폴더명은 `성남에서`, `서초동에서`처럼 `에서`를 유지한다.
- 앱 결과 화면 표시명은 `성남`, `서초동`처럼 `에서`를 제거한다.
- 폴더명과 화면 표시명을 분리했다.

## 오늘 발견/수정한 문제

### 결과 화면에서 바로 정리하기가 실제 정리되지 않는 문제
- 증상:
  - 팝업에서 `결과 보기`로 이동한 뒤 `바로 정리하기`를 눌러도 실제 이동/복사가 안 되는 것처럼 보임.
  - 뒤로 가면 다시 `앨범 정리 시작`을 눌러야 하는 흐름.
- 수정:
  - `startCopyFromPreviewContext()` 추가.
  - `startCopyFromResultScreen()` 추가.
  - 결과 화면에서는 메인 UI 복원 후 `copyButton.post(this::runCopy)`로 실행.
  - 미리보기 완료 후 메인에 `바로 정리하기` 버튼이 보이도록 `copyButton.setVisibility(...)` 처리.

### 정리 완료 후 중복으로 남는 흐름
- 정리 완료 후 자동 결과 화면으로 이동하게 했다.
- 결과 화면에 `사진 원본 삭제`를 표시해서 복사된 사진 원본 삭제로 이어지게 했다.
- `copyCompletedMode` 플래그 추가:
  - 정리 완료 후 `바로 정리하기`를 다시 보여주지 않음.
  - 새 미리보기 또는 폴더 변경 시 초기화.

## 다음 채널에서 꼭 확인할 것
1. 팝업에서 `결과 보기` → 결과 화면 `바로 정리하기` → 실제 정리 실행되는지.
2. 미리보기 후 팝업을 닫거나 뒤로 갔을 때 메인에 `바로 정리하기` 버튼이 보이는지.
3. 정리 완료 후 결과 화면에서 `사진 원본 삭제`가 보이는지.
4. 정리 완료 후 `바로 정리하기`가 숨겨지는지.
5. `갤러리 열기`가 메인/결과 화면에서 모두 동작하는지.
6. 동영상 이동이 실제 기기에서 계속 잘 되는지.

## 현재 제품 방향
- 기능 중심 UI가 아니라 사용자 목표 중심 Flow.
- 사용자는 `앨범 정리 시작` 하나만 이해하면 된다.
- `미리보기`, `정리 실행`, `결과 보기`는 하나의 흐름으로 연결되어야 한다.
- `사진 원본 삭제`만 위험 액션으로 분리한다.
- 개발자 용어보다 사용자 말투 우선:
  - `분류`보다 `정리`
  - `후보`보다 `새로 만들 앨범`
  - `복사본 있음`보다 `이미 앨범에 있는 항목`
