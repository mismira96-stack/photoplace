# AlbumPlace V1 Follow-up - 2026-06-21

> Final source of truth: `WORKLOG_2026-06-21.md`. This file contains intermediate implementation notes.

## Implemented

- Detail Back navigation now returns to the sorting-history list instead of Home.
- The sorting-history `ScrollView` position is saved before opening a detail and restored when returning.
- The detail CTA now opens the matching Samsung Gallery album using:
  - action `com.android.gallery.action.SHORTCUT_ALBUM_VIEW`
  - component `GalleryExternalActivity`
  - MediaStore `BUCKET_ID` as integer extra `ALBUM_ID`
  - `IS_VIRTUAL_ALBUM=false`
- If Samsung direct-album launch is unavailable or fails, the app shows a short notice and opens the normal gallery.
- Added one optional local memory line per generated album.
- Added an optional display alias without renaming the real folder or Samsung Gallery album.
- The detail editor supports save, update, clearing the alias, and deleting the memory line.
- Alias and memory are stored in `SharedPreferences`, keyed by the stable album `relativePath`.
- Sorting-history cards show the alias first and one-line memory preview when present.
- Long title/memory text is ellipsized to protect card layout.

## Device Regression Checklist

1. Open sorting history and scroll down.
2. Open a detail, then use both the top Back and system Back.
3. Confirm the list and prior scroll position return.
4. Add an alias and memory, restart the app, and confirm both persist.
5. Edit and delete the memory.
6. Confirm the alias appears in list/detail while the original album label remains visible in detail.
7. Open a mixed image/video album in Samsung Gallery.
8. Delete or empty an album and confirm the CTA falls back without a crash.

## V1 Release Blocker Fixes - versionCode 15

- Completed sorting results are dismissed after the user-approved original-photo trash operation. Persistent sorting history remains available.
- Fixed album month ranges when `MediaStore.DATE_TAKEN` is missing.
  - Existing-album fallback: EXIF/video metadata, filename date, modified date, then added date.
  - Only rows without `DATE_TAKEN` read embedded metadata to limit performance impact.
- New source analysis now prefers embedded EXIF/video capture time over dates inferred from filenames.
- Verified on the connected Galaxy device:
  - `Grantville에서`: `2019.12~2020.1` corrected to `2019.12`.
  - `Melbourne에서`: corrected to `2019.12` through the same fallback path.
